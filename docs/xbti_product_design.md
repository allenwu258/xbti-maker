# XBTI 测试设计工作台产品设计

版本: 0.1  
日期: 2026-04-20  
项目定位: 面向创作者、运营团队和品牌方的「梗系人格测试」生产工具

## 1. 背景与机会

SBTI 的爆发说明，用户并不只需要严肃人格测评，也需要一种低门槛、强表达、可截图传播的自我标签游戏。根据本地 `C:\Users\Trivedi\sbti_analysis.md` 拆解，SBTI 的可复用骨架是:

- 用 5 组心理/行为模型承载 15 个子维度。
- 每个维度配置 2 道正式题，选项映射为 1/2/3 分。
- 将维度得分离散为 L/M/H，再与人格模板做向量距离匹配。
- 通过隐藏题、兜底人格、荒诞命名、结果长文，制造传播记忆点。
- 题目顺序打乱，少量 gate 题制造彩蛋。

外部页面也在强化同一模式: 30 题、5 大模型、15 维度、约 3-5 分钟完成、即时结果、隐藏人格、结果截图传播，已经成为用户对 SBTI 类玩法的默认预期。

XBTI Maker 不应只是「做一个 SBTI 编辑器」，而应抽象成「主题化测试生产线」: 用户输入一个主题，比如职场、恋爱、猫狗、城市、游戏、品牌活动、二次元角色、咖啡人格，系统自动生成一整套可编辑、可评分、可预览、可导出的测试产品。

## 2. 产品一句话

XBTI Maker 是一个 AI 驱动的趣味人格测试设计工作台，能把任意主题快速转化为「维度模型、测试题、评分算法、人格结果、演示网页和可独立部署 HTML」。

## 3. 目标用户

### 3.1 核心用户

- 内容创作者: 需要追热点、做互动内容、产出可分享的测试页。
- 社媒运营: 需要为节日、活动、品牌话题批量生产测试。
- 品牌市场团队: 需要低成本制作 H5 活动、私域裂变工具、门店互动测试。
- 编辑/策划团队: 需要把脑洞变成结构化题库和结果体系。

### 3.2 间接用户

- 测试参与者: 只关心好不好笑、准不准、是否适合截图分享。
- 开发/交付人员: 只关心导出包是否可上线、可嵌入、可二次开发。

## 4. 产品目标

### 4.1 业务目标

- 把单个主题测试的生产周期从 1-3 天压缩到 10-30 分钟。
- 支持批量生产多版本测试，覆盖热点、节日、品牌活动。
- 输出可直接演示和交付的 HTML 成品，降低研发依赖。
- 形成测试模板库、风格库、结果人设库，越用越快。

### 4.2 用户体验目标

- 新手输入一个主题就能得到完整初稿。
- 专业用户可以细调模型、权重、题目、结果与规则。
- 结果页有传播感，不只是冷冰冰的分数。
- 导出的 HTML 能脱离平台独立运行。

### 4.3 质量目标

- 每个测试有清晰的维度定义、题目映射和结果解释。
- 题目数量、选项方向、反向题、隐藏规则可审计。
- AI 内容可控，避免侵权、歧视、辱骂、医疗诊断化表述。
- 评分算法透明，可复现、可调试、可版本管理。

## 5. 产品边界

### 5.1 做什么

- 主题输入与 AI 一键生成。
- 测试结构设计: 维度、题目、选项、结果人格、彩蛋规则。
- 评分算法配置与模拟。
- 在线预览演示网页。
- 分享卡与结果页文案生成。
- 单文件 HTML 导出，含运行时、样式、题库、评分逻辑。
- 项目版本管理、模板复用、批量改写。

### 5.2 不做什么

- 不宣称提供临床心理诊断。
- 不直接复制 SBTI 原题、原结果文案和人格设定。
- 不做重后端的问卷 SaaS 大而全系统作为第一阶段目标。
- 不把 AI 结果不经审核地自动公开发布。

## 6. 核心使用场景

### 场景 A: 热点跟进

运营输入「打工人精神状态 XBTI」，选择「抽象、嘴毒但温柔、适合小红书/朋友圈传播」，系统生成 30 道题、24 种人格、结果页和分享卡。运营改几处梗后导出 HTML，部署到活动页。

### 场景 B: 品牌活动

咖啡品牌输入「你是哪种咖啡人格」，选择「轻松、都市、适合门店扫码」。系统生成偏消费场景的维度，如能量来源、社交浓度、风味偏好、工作节奏、情绪补给，并把结果人格映射到产品推荐。

### 场景 C: 社群互动

社群管理员输入「我们群友的发疯类型」，选择「内部梗、不可公开传播」。系统生成私域版测试，开启免责声明、隐藏敏感词替换和短链分享。

### 场景 D: 专业策划打磨

策划从 AI 初稿进入算法实验室，调整维度权重、增加 gate 题、设置隐藏人格，使用模拟用户批量跑分，检查是否有结果过度集中。

- 导出发布: 单 HTML、静态资源包、JSON 配置、嵌入代码。

## 8. 关键功能设计

### 8.1 主题 Brief

用户输入:

- 主题: 例如「奶茶人格测试」。
- 目标人群: 例如 18-28 岁大学生/白领。
- 传播平台: 小红书、朋友圈、B 站、微博、私域。
- 语气: 抽象、温柔、毒舌、赛博、乙女、职场黑话、品牌友好。
- 边界: 禁用词、品牌词、不能冒犯的人群、是否允许脏话谐音。
- 规模: 题目数、结果数、维度数、完成时长。
- 目的: 纯娱乐、品牌种草、社群破冰、用户调研、活动裂变。

系统输出:

- 主题拆解
- 世界观设定
- 用户会分享的理由
- 测试结构建议
- 3-5 套命名方向
- 风险提醒

### 8.2 AI 一键生成

一键生成不是直接吐一大坨内容，而是分阶段生成，便于审核和回滚:

1. 生成主题世界观: 这个测试在调侃什么、安慰什么、让用户认领什么。
2. 生成维度模型: 3-6 个模型组，每组 2-4 个子维度。
3. 生成结果人格矩阵: 每个人格有代码、中文名、核心矛盾、传播句。
4. 生成评分模板: 每个人格对应 L/M/H 或连续向量。
5. 生成题库: 每个维度至少 2 题，含场景题和自评题。
6. 生成结果内容: 一句话、长文、优势、风险、朋友相处指南、分享文案。
7. 生成视觉方向: 色彩、字体气质、卡片布局、图标/插画提示词。
8. 生成演示页面: 自动填充起始页、答题页、结果页。

AI 输出需要带结构化依据:

- 每道题为什么绑定该维度。
- 每个选项为什么对应该分值。
- 每个人格模板为什么这样设置。
- 哪些内容可能过火，需要人工确认。

### 8.3 维度模型编辑器

维度是测试可信度的骨架。每个维度应包含:

- 维度 ID: 如 `S1`、`E2`，或主题化 ID 如 `Tea1`。
- 名称: 如「能量补给方式」。
- 所属模型组: 如「情绪模型」。
- 低/中/高定义: L/M/H 各一句解释。
- 评分方向: 分值越高代表越强，或反向。
- 权重: 默认 1，可对核心维度加权。
- 题目覆盖数: 推荐至少 2。
- 结果解释片段: 用于结果页自动拼接。

质量检查:

- 是否有未绑定题目的维度。
- 是否存在所有结果都同一档的无效维度。
- 是否存在定义模糊、正负方向冲突。
- 是否存在过度价值判断，如把低分写成失败者。

### 8.4 题库编辑器

题目类型:

- 单选量表题: 3-5 个选项，各自映射分值。
- 情境选择题: 更适合传播和趣味。
- 反向题: 减少用户机械作答。
- Gate 题: 触发隐藏题或特殊结果。
- 注意力题: 可选，用于识别乱选。
- 分支题: 根据前题答案显示。

单题字段:

- 题目 ID
- 题干
- 题型
- 所属维度
- 选项文本
- 选项分值
- 是否反向
- 是否参与正式计分
- 展示条件
- 标签: 搞笑、尖锐、温柔、品牌、敏感
- AI 解释

编辑能力:

- 一键改写语气
- 批量缩短题干
- 批量增加抽象感
- 生成更多同维度候选题
- 检测题目是否太像严肃诊断
- 检测同一维度题目是否重复

### 8.5 结果人格编辑器

每个结果人格包含:

- 代码: 3-6 位，便于传播，如 `CTRL`、`GOGO`。
- 中文名: 可记忆，可截图传播。
- 类型标签: 标准、稀有、隐藏、兜底、品牌推荐。
- 模板向量: 如 `HHM-LMH-MMM...`。
- 一句话: 适合截图顶部。
- 长文: 300-800 字，可分段。
- 维度解释: 自动引用高/低维度。
- 优势/雷点/相处指南。
- 分享标题: 用于社媒转发。
- 视觉资产: 头像、图标、背景、主色。
- 推荐动作: 品牌商品、内容链接、社群入口。

人格设计原则:

- 每个人格都要有「可认领的羞耻感」和「被理解的补偿感」。
- 不只骂用户，要在最后给一点温柔回收。
- 名称要短、好念、可二创。
- 人格之间差异应来自维度矛盾，而不是随机段子堆砌。

### 8.6 评分算法实验室

#### 默认算法: L/M/H 模板匹配

1. 统计每个维度总分。
2. 将总分映射为 L/M/H。
3. 将用户向量与结果人格模板逐一计算距离。
4. 选择距离最小的人格。
5. 距离相同则按精确命中数、权重命中数、优先级排序。
6. 命中隐藏规则时可覆盖标准结果。
7. 低置信度时进入兜底人格或展示「混合人格」。

推荐公式:

```text
dimension_score[d] = sum(option_score for questions bound to d)
level[d] = L if score <= low_threshold, M if score <= mid_threshold, else H
distance(user, archetype) = sum(weight[d] * abs(level_value[user[d]] - level_value[archetype[d]]))
similarity = round((1 - distance / max_distance) * 100)
```

#### 可选算法

- 连续向量匹配: 不离散为 L/M/H，直接使用 0-100 分。
- Top-N 混合人格: 输出主人格 + 副人格。
- 规则优先算法: gate 题、极端维度、品牌推荐优先。
- 分组算法: 先判定大类，再判定小类。
- 百分位算法: 有真实数据后按人群分布校准。

#### 算法调试视图

- 用户每题答案
- 每维度原始分
- L/M/H 映射
- 各人格距离
- Top 5 候选
- 触发规则
- 最终选择原因

### 8.7 规则和彩蛋

规则类型:

- Gate 题触发隐藏人格。
- 某维度极高/极低触发稀有人格候选。
- 特定选项组合触发彩蛋文案。
- 结果相似度低于阈值触发兜底人格。
- 品牌活动中，特定结果映射不同优惠或推荐。

规则编辑方式:

```text
IF question.drink_gate_q1 == "饮酒"
THEN show question.drink_gate_q2

IF question.drink_gate_q2 == "重度饮酒"
THEN result = DRUNK
```

系统必须展示规则优先级，避免多个隐藏结果同时触发时不可解释。

### 8.8 模拟校准

校准目标:

- 避免 70% 用户都落到同一人格。
- 避免某些人格永远无法命中。
- 避免隐藏人格过易触发。
- 检查题目分值方向是否写反。
- 检查结果文案是否与维度不一致。

模拟方式:

- 随机答题 10,000 次。
- 偏高/偏低用户模拟。
- 极端用户模拟。
- 手动创建样本用户。
- 导入真实答题 CSV 后复盘。

输出指标:

- 人格命中分布
- 维度分布
- 平均完成时长
- 题目跳出率
- Top 结果相似度均值
- 隐藏结果触发率
- 题目区分度

### 8.9 页面搭建与预览

页面结构:

- 起始页: 标题、副标题、主题说明、免责声明、开始按钮。
- 答题页: 进度、题目、选项、返回、随机题序、分支显示。
- 结果页: 人格名、匹配度、短句、长文、维度雷达/条形图、分享按钮。
- 人格图鉴页: 可选，用于活动站。
- 分享卡: 生成竖图，适配小红书/朋友圈。

预览模式:

- 手机宽度
- 桌面宽度
- 暗色/亮色
- 从任意题开始预览
- 指定人格结果预览
- 随机用户完整走查

### 8.10 HTML 导出

导出目标是「可独立运行」。MVP 优先支持单文件 HTML:

- 内嵌 CSS
- 内嵌 JS 运行时
- 内嵌测试 JSON
- 无后端依赖
- 本地打开可运行
- 可部署到任意静态托管

高级导出:

- 静态资源包: `index.html` + `assets/` + `test-config.json`
- 嵌入版: 提供 iframe 或 script embed
- 数据采集版: 可配置 webhook/API 上报
- 多语言版: 导出语言包
- 白标版: 去除平台标识

导出前检查:

- 是否存在未完成题目。
- 是否存在无结果人格。
- 是否存在不可达结果。
- 是否包含敏感词。
- 是否有心理诊断化风险。
- 是否通过手机预览。

## 9. AI 生成体验设计

### 9.1 生成模式

- 快速模式: 输入主题，一键生成完整可演示版本。
- 专业模式: 分阶段确认维度、题目、结果、页面。
- 批量模式: 输入多个主题，批量生成草稿。
- 改写模式: 对已有测试换主题、换语气、换平台。

### 9.2 AI 助手能力

- 「帮我想 20 个更好传播的人格名」
- 「这套题太严肃，改得更像互联网发疯」
- 「把结果文案改成品牌友好，不要骂人」
- 「补齐每个维度 2 道题」
- 「检查算法是否有不可达人格」
- 「把这个测试改成 10 题极速版」
- 「生成一个适合小红书封面的分享卡文案」

### 9.3 AI 内容约束

系统提示词应内置以下约束:

- 定位为娱乐互动，不输出心理诊断结论。
- 不鼓励自残、成瘾、歧视、骚扰。
- 不对种族、性别、地域、疾病等做贬损人格化。
- 不直接复刻已有测试的受保护文案。
- 对冒犯性昵称做风险标注，交给用户确认。
- 对品牌场景默认更温和。

## 11. 权限与协作

角色:

- Owner: 管理项目、导出、删除。
- Editor: 编辑题库、结果、页面。
- Reviewer: 评论、审批、运行检查。
- Viewer: 只读预览。

协作能力:

- 评论定位到题目/结果/规则。
- 版本差异对比。
- AI 修改前后 diff。
- 审批流: 草稿 -> 待审 -> 可导出 -> 已发布。
- 回滚到历史版本。

## 12. 内容安全与合规

必须明确:

- 测试仅供娱乐、自我表达与社交互动。
- 不应用于医疗、心理诊断、招聘筛选、信贷风控、教育分流。
- 结果描述避免绝对化标签，如「你一定」「你永远」。
- 对负向人格给出情绪缓冲和正向解释。
- 用户数据采集默认关闭，开启时要展示隐私说明。

敏感内容检查:

- 人身攻击
- 歧视性表达
- 未成年人不适内容
- 成瘾行为美化
- 极端消极暗示
- 侵权复刻
- 品牌竞品风险

## 14. MVP 用户流程

1. 用户点击「新建 XBTI」。
2. 输入主题和风格，例如「职场发疯人格，抽象但不恶毒」。
3. 点击「AI 生成完整测试」。
4. 系统生成维度、题库、结果人格、评分模板、页面。
5. 用户进入工作台检查并修改。
6. 用户在预览器里完成一次模拟答题。
7. 系统展示评分过程和最终结果。
8. 用户点击「导出 HTML」。
9. 系统运行导出前检查，通过后生成 `index.html`。
10. 用户本地打开或上传静态托管。

## 20. 主要风险与对策

### 风险 1: 内容同质化

对策: 引入主题世界观、人格矛盾、语气风格、平台风格四层生成约束，避免所有测试都像 SBTI 换皮。

### 风险 2: 题目好笑但算法无效

对策: 强制维度覆盖检查、分值方向检查、模拟校准、不可达人格检测。

### 风险 3: 文案过火引发舆情

对策: 风险词库、敏感人群检测、品牌安全模式、负向人格温柔回收。

### 风险 4: 用户把娱乐测试当专业诊断

对策: 默认免责声明，禁用诊断化措辞，结果页不输出医疗/心理建议。

### 风险 5: 导出后难以维护

对策: 导出 HTML 内嵌版本号和配置 JSON，支持从 HTML 反向导入项目配置。

## 21. 推荐默认模板: XBTI-15

为了快速起步，平台可以内置一个通用模板:

- 5 个模型组
- 15 个子维度
- 每维度 2 题
- 30 道正式题
- 24 个标准人格
- 2 个隐藏人格
- 1 个兜底人格
- L/M/H 三档
- 加权距离匹配

模型组可按主题自动改名:

- 自我模型: 我怎么看自己。
- 关系模型: 我如何靠近别人。
- 世界模型: 我如何理解外部环境。
- 行动模型: 我如何推进事情。
- 表达模型: 我如何被别人看见。

这个模板既继承 SBTI 的传播结构，又不会局限在原 SBTI 的具体题目和人格设定里。

## 22. 结论

XBTI Maker 的核心价值不是「自动写问卷」，而是把一个抽象主题转化为可运行的互动产品。它的关键竞争力由四部分组成:

- 内容生成力: 主题、梗、人格、结果文案能快速成型。
- 结构可信度: 维度、题目、评分和结果之间有可解释关系。
- 交付完整度: 从设计到预览再到 HTML 导出闭环。
- 传播适配度: 结果页、分享卡、人格命名天然适合社交平台。

只要 MVP 把「输入主题 -> AI 初稿 -> 可编辑评分 -> 可预览 -> HTML 导出」这条链路做顺，产品就能满足第一批创作者和运营团队的真实需求。后续再通过模板库、协作、数据回流和商业化能力，升级为专业的测试内容生产平台。
